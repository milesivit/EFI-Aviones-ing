Backend Development Workflow

The backend of this application was developed using Django, following a structured and modular approach. We implemented the complete backend flow which includes:

  • Model: Definition of the database structure.
  • Repository: (optional layer) Encapsulation of database access logic.
  • Service: Implementation of business logic and application rules.
  • View: Definition of the logic to handle HTTP requests and responses.
  • URL: Configuration of routes to connect views with accessible endpoints.

This separation of concerns ensures clean code, easier maintenance, and better scalability.

Data Models and Relationships

The system models cover key entities of an airline application. Here's a brief overview:

  • Plane: Represents aircrafts, with model, capacity, rows and columns.
  • User: System users (e.g. admin, vendors) with login credentials and roles.
  • Flight: Flights with origin, destination, timing, base price, linked to a plane and users.
  • Passenger: Personal information of individuals booking flights.
  • Seat: Specific seats in a plane, including row, column, type and status.
  • Reservation: A passengerâ€™s booking, linked to a flight, seat and passenger, with a unique code.
  • Ticket: Issued ticket for a reservation, with barcode and issue date.

These models are interconnected through ForeignKey and OneToOne relations, ensuring data consistency and efficient retrieval.
